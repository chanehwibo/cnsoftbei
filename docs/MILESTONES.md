# 阶段开发计划

## M1: MVP 闭环

目标：完成自然语言输入、安全判断、工具调用、审计记录的最小可运行闭环。

验收：

- CLI 可以处理系统信息、资源指标、进程、日志、服务状态查询。
- 高风险请求被拒绝。
- 中风险请求需要确认。
- 所有请求写入审计日志。
- 单元测试通过。

状态：已完成初版。

## M2: 麒麟系统适配

目标：在真实麒麟操作系统或虚拟机中验证系统工具。

任务：

- 验证 `/etc/os-release` 对麒麟版本识别是否准确。
- 验证 `systemctl`、`journalctl`、`ps` 在麒麟环境中的输出。
- 增加麒麟包管理、服务管理、网络状态工具。
- 增加命令超时、输出截断和异常归一化。

验收：

- 在麒麟环境完成至少 10 个只读运维场景。
- 所有工具输出稳定 JSON 结构。

## M3: MCP 标准化

目标：将当前 MCP 风格 facade 替换或封装为标准 MCP Server。

任务：

- 接入官方 MCP Python SDK 或实现兼容协议入口。
- 为所有工具补齐 JSON Schema。
- 增加工具调用审计和错误码规范。
- 编写 MCP 客户端调用示例。

验收：

- 大模型客户端可以通过 MCP 发现和调用工具。
- 工具权限和审计逻辑仍由本项目控制。

## M4: Web 运维工作台

目标：提供适合比赛演示的 Web 控制台。

任务：

- 会话区：自然语言请求和 Agent 回复。
- 系统状态区：CPU、内存、磁盘、服务摘要。
- 风险审批区：中风险操作确认，高风险拒绝原因。
- 审计区：最近请求和执行结果。

验收：

- 演示人员可以在 Web 页面完成主要场景。
- 风险提示和审批过程清晰可见。

## M5: 大模型接入

目标：接入可配置大模型，同时保持安全策略在本地强制执行。

任务：

- 支持本地模型或云端模型配置。
- 模型只负责理解和规划，不直接执行系统命令。
- Agent Core 将模型输出转为受控工具调用。
- 增加提示注入和危险意图测试。

验收：

- 自然语言表达更灵活。
- 危险请求仍能被本地策略拦截。

## M6: 答辩材料

目标：完成比赛提交和答辩材料。

任务：

- 系统设计说明书。
- 安全护栏设计文档。
- 测试报告。
- 部署文档。
- 演示脚本和录屏方案。

验收：

- 能完整展示“智能运维 + MCP + 安全护栏 + 审计追踪”。
