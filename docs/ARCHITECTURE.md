# 架构设计

## 1. 目标

SafeOps Agent 在自然语言入口和操作系统之间建立一条受控执行链。模型和规则只负责提出候选工具；本地策略决定是否允许执行；真实操作只能进入注册表中的固定 handler。

## 2. 请求链路

~~~text
CLI / Web / MCP
        |
输入长度与高危意图筛查
        |
LLM 意图理解或离线规则路由
        |
固定工具 + 结构化参数
        |
参数校验、服务边界与三级风险裁决
        |
LOW: 执行
MEDIUM: dry-run + 一次性令牌 -> 再次裁决 -> 执行
HIGH: 拒绝
        |
结果 + 结构化决策轨迹 + HMAC 签名审计
~~~

## 3. 模块

| 模块 | 职责 |
| --- | --- |
| `agent.py` | 会话上下文、路由、决策轨迹、确认流程与结果编排 |
| `llm/` | OpenAI 兼容的 DeepSeek 意图理解与离线规则 Provider |
| `tools/` | 25 个固定工具、诊断聚合和受管文件事务 |
| `security/policy.py` | 意图、参数、服务白名单和风险裁决 |
| `security/pending.py` | 加密保存、会话绑定、限时、一次性确认令牌 |
| `audit/logger.py` | 脱敏、SHA 链、HMAC、锚点、轮转与兼容校验 |
| `mcp_stdio.py` | MCP JSON-RPC 生命周期、Schema 校验和 stdio 传输 |
| `web_server.py` | Cookie 会话认证、限流、SSE、API 和静态资源 |
| `config.py` | 标准 YAML、源码/安装态资源选择和运行目录解析 |

## 4. 状态与数据

- 公共配置位于 `config/`，wheel 内有同内容只读后备资源。
- 私密模型配置仅放在 `config/llm.local.yaml` 或环境变量。
- 运行数据统一位于 `data/`：审计、密钥、锚点、待确认动作、受管文件和快照。
- 待确认动作与快照使用线程锁、跨进程锁和原子替换，避免并发覆盖或半写入。
- Web 每个会话拥有独立 Agent 上下文和锁；会话数有上限并淘汰最久未用项。

## 5. 安全不变量

1. 不存在任意 Shell 工具。
2. 模型不能修改风险等级、确认要求或服务白名单。
3. MEDIUM 动作没有合法一次性令牌就不能执行。
4. 确认执行使用签发时保存的工具和参数，不重新解释原始文本。
5. 保护服务和非白名单服务不能进入生命周期 handler。
6. 审计中的秘密、令牌和文件内容在写盘前脱敏。
7. 已签名审计段缺少 HMAC、链断裂、尾部截断或锚点不符均校验失败。
8. Web 非回环监听必须同时启用 TLS 和认证；明文 HTTP 仅允许回环地址。

## 6. 安装态布局

源码运行优先使用仓库 `config/` 和 `web/`。wheel 安装后，如果工作目录没有这些文件，则从 `safeops_agent/resources/` 读取内置默认配置与静态页面；`PROJECT_ROOT` 指向启动工作目录，运行数据不会写入 site-packages。
