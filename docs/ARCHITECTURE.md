# 架构设计

## 目标

构建一个面向麒麟操作系统的安全智能运维 Agent，作为自然语言和操作系统之间的受控桥梁。

## 模块

- `safeops_agent.agent`: Agent 主流程，负责意图识别、安全判断、工具调用和结果汇总。
- `safeops_agent.tools`: 运维工具集合，只暴露固定能力，不提供任意命令执行入口。
- `safeops_agent.security`: 风险分级、危险意图拦截、审批策略。
- `safeops_agent.audit`: 审计日志，记录请求和执行过程。
- `safeops_agent.mcp_server`: MCP 风格工具清单和工具调用入口。

## 风险分级

- `LOW`: 只读查询，可自动执行。
- `MEDIUM`: 影响局部服务或状态，需要用户确认。
- `HIGH`: 可能造成数据破坏、越权或系统不可用，默认拒绝。

## 安全原则

- 不直接执行模型生成的 shell 命令。
- 每个工具使用固定参数和固定命令模板。
- 工具按风险等级注册。
- 审计日志记录可复盘的决策摘要，不记录隐藏思维链。
